# 捡垃圾（项目代号）

Roblox 单局捡垃圾寻宝游戏。核心玩法参考《老婆随手扔了我多年珍藏卡牌》，玩家在单局对局中翻找垃圾、变现、成长，稀有物首次获得后作为永久收藏留在家园展示。

## 新对话开工前，按顺序看这三样

1. 本文件（当前阶段是什么）
2. `开发记录.md` 最近 2-3 条（不用翻全部历史）
3. `git status`（看当前工作区有没有未提交的改动）

只有任务明确涉及架构或 Remote 时，才去读 `项目设定相关/架构与Remote登记.md`；不需要每次都全量读完所有设计文档。

## 本地代码目录（Script Sync）

项目用 Roblox Studio 官方 **Script Sync** 做代码本地化和版本管理（不用 Rojo）。仓库根目录下 `home/` 对应家园 Place，`battle/` 对应对局 Place，两边分别在 Studio 里配置 Sync to 绑定。详细操作步骤和限制见《基础技术框架.md》第7节。

**注意**：Script Sync 只同步 Script/LocalScript/ModuleScript/Folder，场景搭建、Tag/Attribute、RemoteEvent 实例等非脚本内容不会进 git，仍然只存在于 Studio 里，改动后要写进《开发记录.md》。

## 硬规则（不为省事而破坏）

1. 涉及钱、掉落判定、稀有物解锁的逻辑，必须服务端权威，客户端只管手感和展示。
2. 新增/修改 RemoteEvent，先更新 `架构与Remote登记.md` 再写代码。
3. 每次任务对应一次干净的 git 提交，不混无关改动；有未提交的改动先处理干净再开始新任务。
4. 结论要标注清楚是"已验证"还是"推断"，没验证过的不能包装成事实。
