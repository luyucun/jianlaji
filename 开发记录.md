# 开发记录

记录每次开发/文档改动的核心结果，方便回滚和新对话接手。不写过程流水账，只写结论。

新记录写在最上面（倒序），最新的最容易被看到。

记录模板：

```md
## YYYY-MM-DD 简短标题
- 改了什么：一句话
- 影响文件：列出主要文件
- 验证情况：验证过什么 / 未验证的原因
- Git 提交：`hash 提交说明`（未提交则写"未提交"）
```
