# 策划案

## V1.1 基础的场景切换

1.玩家在场中处于Workspace - Base - RoundEntrances - Entrance_Solo所在范围内后，开始5秒倒计时，倒计时结束后，将玩家传送至战斗场景。先做这个流程即可，其他都不用做



#### V1.1.1 补充一些切换场景相关的细节

1.我们现在靠场中的Entrance_Solo来决定是否传送，我们需要支持场中如果有多个Entrance_Solo，多个玩家各自使用Entrance_Solo，都能成功传送；

2.Entrance_Solo - Tips - State - TextLabel是用来显示当前的准备状态，默认状态在文本是0/1，但是如果已经有人进来了，那么需要改成1/1,同时把文本颜色改成绿色；

3.你需要处理好一个问题，一个Entrance_Solo被一个玩家占据后，其他玩家就无法再进来站上来了，所以需要围绕着Entrance_Solo周围给一圈透明空气墙，不让其他玩家再进来

4.里面的玩家可以在倒计时结束之前，随时离开Entrance_Solo，离开后，传送倒计时结束，解除对当前这个Entrance_Solo的占位，其他人就可以继续上来了

5.当一个玩家传送完成后，这个Entrance_Solo也解除占用，可以供下一个玩家使用

6.当进入倒计时后，需要在玩家的界面上显示出来：StarterGui - Main - LeaveButton显示出来，其中：

- 玩家点击LeaveButton - Leave按钮，强制将玩家移除出这个Entrance_Solo范围，放到边上；

- LeaveButton - CoundownTime是进入倒计时，内容就是xS，x是剩余的倒计时秒数数字



### V1.2 基础的垃圾堆生成

概述：我们这个版本要做的是第一版的基础的垃圾堆生成的逻辑

详细规则：

1.在Battle这个场景中，当玩家进入后，Workspace - TrashArea是我们用来生成垃圾堆的路径

2.TrashArea下会有多个Part，分别叫Area1到AreaN，我们会在我们的数据表中配置，每个Area下应该要生成的垃圾的数量，也就是每个Area就是一个垃圾堆，至于这个垃圾堆里要多少数量的垃圾，我们在表中配置

3.ReplicatedStorage - Trash 下存放着我们的垃圾模型，我们会在表中添加每个垃圾出现的权重，以及所出现的垃圾堆的区域

4.垃圾生成时，要能够生成一堆，尽量不要超出Area 的区域，这些垃圾之间互相有碰撞，与玩家也有碰撞，垃圾要做出来堆在一起的效果

5.我们这个版本先做出垃圾堆生成的逻辑



我们的数据表放在：项目设定相关，这个文件夹下
